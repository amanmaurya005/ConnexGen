# Project Uses – Social Media Application

This document describes the **technologies, tools, and libraries** used in building the full-stack social media application.

---

## 1. Frontend Technologies

### 1.1 Core
- **React.js** – Component-based UI development
- **JavaScript (ES6+)** – Application logic
- **HTML5** – Markup structure

### 1.2 UI & Styling
- **Tailwind CSS / CSS Modules** – Responsive and modern UI design

### 1.3 State & Routing
- **React Hooks** (`useState`, `useEffect`, `useContext`)
- **React Router DOM** – Client-side routing
- **Context API** – Global state management

### 1.4 Data Handling
- **Axios / Fetch API** – API communication
- **Infinite Scroll** – Load posts dynamically while scrolling
- **Debouncing** – Optimized search functionality

---

## 2. Backend Technologies

### 2.1 Server & Framework
- **Node.js** – JavaScript runtime
- **Express.js** – REST API development

### 2.2 Authentication & Security
- **JWT (JSON Web Token)** – User authentication
- **bcrypt.js** – Password hashing
- **Middleware** – Route protection and validation
- **CORS** – Secure cross-origin requests

---

## 3. Database

### 3.1 Database
- **MongoDB** – MongoDB database for storing all application data

### 3.2 ODM
- **Mongoose** – Schema modeling and database interaction

### 3.3 Stored Data
- Users
- Posts
- Comments
- Likes
- Followers / Following
- Messages

---

## 4. Messaging & Real-Time (Optional)

- **Socket.IO** – Real-time messaging
- **WebSockets** – Live chat communication

---

## 5. Features Implemented Using Tech Stack

- User authentication & authorization

- Private messaging system

## 9. Why MongoDB?

- Flexible schema design
- High scalability
- Fast read/write operations
- Perfect for social media–style data
- Easy integration with Node.js

## 10. Conclusion

This project uses a **modern MERN-style stack** to build a scalable and production-ready social media application with full database integration and real-time capabilities.
